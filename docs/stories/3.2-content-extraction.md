# Story 3.2: Content Extraction & JSON Structure

**Epic:** AITELIER-3.0-I18N
**Status:** Draft
**Priority:** P0
**Estimate:** 4-6 hours
**Depends on:** 3.1

---

## Story

**As a** developer,
**I want** all hardcoded strings extracted to JSON translation files,
**so that** the site content is translatable and maintainable.

---

## Acceptance Criteria

1. All navigation items extracted to `navigation.json`
2. All page content extracted to respective JSON files
3. All form labels/placeholders/messages extracted to `forms.json`
4. All button texts extracted to `common.json`
5. All error/success messages extracted
6. Components using `useTranslation()` hook
7. No hardcoded Portuguese strings remaining in components
8. JSON structure is logical and developer-friendly

### JSON Structure Pattern
```json
{
  "sectionName": {
    "title": "Section Title",
    "description": "Description text",
    "items": {
      "item1": "First item",
      "item2": "Second item"
    }
  }
}
```

### Component Update Pattern
```typescript
// Before
<p>A arte que ensinamos cria.</p>

// After
import { useTranslation } from 'react-i18next';

const { t } = useTranslation('home');
<p>{t('hero.line1')}</p>
```

### Finding Hardcoded Strings
```bash
# Search for Portuguese characters in TSX files
grep -r "[áéíóúãõâêîôûç]" src/**/*.tsx
```

### Key Naming Convention
- Use camelCase for keys
- Nest by section/component
- Keep keys semantic, not positional
- Example: `hero.tagline` not `hero.line1`

### Priority Order for Extraction
1. Navigation (visible on all pages)
2. Footer (visible on all pages)
3. Home page sections
4. Contact form
5. Other pages
